# 상세페이지 기본 정보 섹션 수동 확인 체크리스트

> 서버 실행 후 (`pnpm dev`) 손으로 직접 확인하는 체크리스트입니다.

## 📋 기본 확인 사항

### 1. 홈페이지에서 상세페이지로 이동 확인

- [ ] **홈페이지 접속**: `http://localhost:3000` 접속
- [ ] **관광지 목록 확인**: 관광지 카드들이 표시되는지 확인
- [ ] **카드 클릭**: 아무 관광지 카드나 클릭
- [ ] **상세페이지 이동**: `/places/[숫자]` 형태의 URL로 이동하는지 확인
- [ ] **페이지 로딩**: 상세페이지가 정상적으로 로드되는지 확인

---

### 2. 상세페이지 기본 정보 표시 확인

#### 2.1 관광지명
- [ ] **대제목 표시**: 페이지 상단에 관광지명이 크게 표시되는지 확인
- [ ] **제목 스타일**: 제목이 잘 보이고 읽기 쉬운지 확인

#### 2.2 대표 이미지
- [ ] **이미지 표시**: 관광지 대표 이미지가 크게 표시되는지 확인
- [ ] **이미지 비율**: 이미지가 가로로 길쭉하게 표시되는지 확인 (16:9 비율)
- [ ] **이미지 없을 때**: 이미지가 없는 관광지의 경우, 회색 배경에 이미지 아이콘이 표시되는지 확인
- [ ] **이미지 로드 실패**: 이미지 URL이 잘못된 경우에도 플레이스홀더가 표시되는지 확인

#### 2.3 관광 타입 뱃지
- [ ] **뱃지 표시**: 관광지명 아래에 "관광지", "문화시설", "음식점" 등의 뱃지가 표시되는지 확인
- [ ] **뱃지 색상**: 뱃지가 파란색 계열로 표시되는지 확인

#### 2.4 주소 정보
- [ ] **주소 표시**: 관광지 주소가 표시되는지 확인
- [ ] **주소 형식**: "시/도 시/군/구 상세주소" 형태로 표시되는지 확인
- [ ] **복사 버튼**: 주소 옆에 "복사" 버튼이 있는지 확인
- [ ] **복사 기능**: "복사" 버튼 클릭 시
  - [ ] 화면 상단에 "주소가 복사되었습니다." 메시지가 나타나는지 확인
  - [ ] 버튼이 "복사됨"으로 바뀌는지 확인 (체크 아이콘 표시)
  - [ ] 2초 후 다시 "복사"로 돌아오는지 확인
- [ ] **복사 내용 확인**: 다른 곳(메모장 등)에 붙여넣기 해서 주소가 제대로 복사되었는지 확인

#### 2.5 전화번호
- [ ] **전화번호 표시**: 전화번호가 있는 관광지의 경우 전화번호가 표시되는지 확인
- [ ] **전화 아이콘**: 전화번호 옆에 전화 아이콘이 있는지 확인
- [ ] **전화번호 클릭**: 전화번호를 클릭하면
  - [ ] 모바일: 전화 앱이 열리는지 확인
  - [ ] 데스크톱: 전화 앱 또는 기본 전화 프로그램이 열리는지 확인
- [ ] **전화번호 없을 때**: 전화번호가 없는 관광지의 경우 해당 항목이 표시되지 않는지 확인

#### 2.6 홈페이지
- [ ] **홈페이지 표시**: 홈페이지가 있는 관광지의 경우 홈페이지 URL이 표시되는지 확인
- [ ] **외부 링크 아이콘**: 홈페이지 옆에 외부 링크 아이콘이 있는지 확인
- [ ] **홈페이지 클릭**: 홈페이지를 클릭하면
  - [ ] 새 창(탭)에서 홈페이지가 열리는지 확인
  - [ ] 현재 페이지는 그대로 유지되는지 확인
- [ ] **홈페이지 없을 때**: 홈페이지가 없는 관광지의 경우 해당 항목이 표시되지 않는지 확인

#### 2.7 개요
- [ ] **개요 표시**: 관광지에 대한 긴 설명문이 표시되는지 확인
- [ ] **줄바꿈**: 설명문이 여러 줄로 표시되는지 확인 (줄바꿈이 제대로 되는지)
- [ ] **텍스트 스타일**: 설명문이 읽기 쉬운 크기와 색상으로 표시되는지 확인
- [ ] **개요 없을 때**: 개요가 없는 관광지의 경우 해당 항목이 표시되지 않는지 확인

---

### 3. 뒤로가기 버튼 확인

- [ ] **뒤로가기 버튼**: 페이지 상단 왼쪽에 "뒤로가기" 버튼이 있는지 확인
- [ ] **뒤로가기 아이콘**: 버튼 옆에 왼쪽 화살표 아이콘이 있는지 확인
- [ ] **뒤로가기 클릭**: "뒤로가기" 버튼을 클릭하면
  - [ ] 홈페이지(`/`)로 이동하는지 확인
  - [ ] 이전에 보던 관광지 목록이 그대로 있는지 확인

---

### 4. 반응형 디자인 확인

#### 4.1 모바일 화면 (브라우저 개발자 도구 사용)
- [ ] **브라우저 개발자 도구 열기**: `F12` 또는 `우클릭 → 검사`
- [ ] **모바일 모드 전환**: 개발자 도구에서 모바일 아이콘 클릭 (또는 `Ctrl+Shift+M`)
- [ ] **화면 크기**: 375px × 667px (iPhone SE) 크기로 설정
- [ ] **레이아웃 확인**:
  - [ ] 관광지명이 한 줄로 표시되는지 확인
  - [ ] 이미지가 화면 너비에 맞게 표시되는지 확인
  - [ ] 주소와 복사 버튼이 세로로 배치되는지 확인
  - [ ] 전화번호와 홈페이지가 세로로 배치되는지 확인
  - [ ] 텍스트가 잘리지 않고 모두 보이는지 확인

#### 4.2 태블릿 화면
- [ ] **화면 크기**: 768px × 1024px (iPad) 크기로 설정
- [ ] **레이아웃 확인**:
  - [ ] 주소와 복사 버튼이 가로로 배치되는지 확인
  - [ ] 전화번호와 홈페이지가 가로로 배치되는지 확인
  - [ ] 여백과 간격이 적절한지 확인

#### 4.3 데스크톱 화면
- [ ] **화면 크기**: 1920px × 1080px 크기로 설정
- [ ] **레이아웃 확인**:
  - [ ] 콘텐츠가 중앙에 배치되는지 확인
  - [ ] 최대 너비가 제한되어 있는지 확인 (너무 넓지 않음)
  - [ ] 모든 정보가 한 눈에 잘 보이는지 확인

---

### 5. 에러 처리 확인

#### 5.1 잘못된 관광지 ID
- [ ] **URL 직접 입력**: 브라우저 주소창에 `http://localhost:3000/places/abc123` 입력
- [ ] **에러 메시지**: "잘못된 관광지 ID입니다. contentId는 숫자만 허용됩니다." 메시지가 표시되는지 확인
- [ ] **에러 아이콘**: 빨간색 경고 아이콘이 표시되는지 확인

#### 5.2 존재하지 않는 관광지 ID
- [ ] **URL 직접 입력**: 브라우저 주소창에 `http://localhost:3000/places/999999999` 입력 (존재하지 않는 ID)
- [ ] **에러 메시지**: "관광지 정보를 불러오는 중 오류가 발생했습니다." 또는 유사한 에러 메시지가 표시되는지 확인
- [ ] **에러 아이콘**: 빨간색 경고 아이콘이 표시되는지 확인

---

### 6. 다양한 관광지 타입 확인

다양한 타입의 관광지를 확인하여 모든 정보가 제대로 표시되는지 확인:

- [ ] **관광지 (12)**: 일반 관광지 확인
- [ ] **문화시설 (14)**: 박물관, 미술관 등 확인
- [ ] **음식점 (39)**: 음식점 확인
- [ ] **숙박 (32)**: 호텔, 펜션 등 확인
- [ ] **기타 타입**: 다른 타입의 관광지도 확인

각 타입별로:
- [ ] 관광 타입 뱃지가 올바르게 표시되는지 확인
- [ ] 모든 정보 항목이 제대로 표시되는지 확인

---

### 7. 정보 없는 항목 숨김 처리 확인

다음 정보가 없는 관광지를 찾아서 확인:

- [ ] **이미지 없음**: 이미지가 없는 관광지에서 플레이스홀더가 표시되는지 확인
- [ ] **전화번호 없음**: 전화번호 항목이 표시되지 않는지 확인
- [ ] **홈페이지 없음**: 홈페이지 항목이 표시되지 않는지 확인
- [ ] **개요 없음**: 개요 항목이 표시되지 않는지 확인

---

### 8. 키보드 네비게이션 확인 (접근성)

- [ ] **Tab 키**: `Tab` 키를 눌러서 모든 버튼과 링크에 포커스가 이동하는지 확인
- [ ] **포커스 스타일**: 포커스된 요소에 파란색 테두리(ring)가 표시되는지 확인
- [ ] **Enter 키**: 포커스된 버튼에서 `Enter` 키를 누르면 동작하는지 확인
- [ ] **주소 복사**: `Tab` 키로 주소 복사 버튼에 포커스 후 `Enter` 키로 복사가 되는지 확인

## ✅ 빠른 확인 체크사항 (5분 안에 확인)

서버 실행 후 빠르게 확인할 수 있는 핵심 사항들:

1. **홈페이지 → 상세페이지 이동**
   - [ ] 관광지 카드 클릭 → 상세페이지로 이동하는지 확인

2. **기본 정보 표시**
   - [ ] 관광지명이 크게 표시되는지 확인
   - [ ] 대표 이미지가 표시되는지 확인
   - [ ] 주소가 표시되는지 확인
   - [ ] 관광 타입 뱃지가 표시되는지 확인

3. **주소 복사 기능**
   - [ ] "복사" 버튼 클릭 → "주소가 복사되었습니다." 메시지가 나타나는지 확인

4. **뒤로가기 버튼**
   - [ ] "뒤로가기" 버튼 클릭 → 홈페이지로 돌아가는지 확인

5. **반응형 디자인**
   - [ ] 브라우저 창 크기를 줄여서 모바일처럼 보이게 했을 때 레이아웃이 깨지지 않는지 확인

6. **에러 처리**
   - [ ] 잘못된 URL (`/places/abc123`) 입력 → 에러 메시지가 표시되는지 확인

## 📝 확인 시 주의사항

1. **서버 실행**: 반드시 `pnpm dev`로 개발 서버를 실행한 후 확인하세요.
2. **브라우저 캐시**: 문제가 있을 경우 브라우저 캐시를 지우고(`Ctrl+Shift+Delete`) 다시 확인하세요.
3. **네트워크**: API 호출이 실패할 수 있으므로, 네트워크 연결 상태를 확인하세요.
4. **환경변수**: `.env` 파일에 `NEXT_PUBLIC_TOUR_API_KEY` 또는 `TOUR_API_KEY`가 설정되어 있는지 확인하세요.

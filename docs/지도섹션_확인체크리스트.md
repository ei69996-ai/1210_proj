# 지도 섹션 (MVP 2.4.4) 수동 테스트 체크리스트

> 상세페이지 지도 섹션 기능이 정상적으로 작동하는지 확인하는 체크리스트입니다.
> 서버 실행 후 (`pnpm dev`) 아래 항목들을 하나씩 확인해주세요.

---

## 1. 기본 표시 확인

### 1.1 지도 섹션이 보이는지 확인
- [ ] 상세페이지에서 "위치" 섹션이 표시되는지 확인
- [ ] 지도가 정상적으로 로드되는지 확인 (회색 배경이 아닌 실제 지도)
- [ ] 지도 위에 빨간색 핀(마커)이 표시되는지 확인

**확인 방법:**
1. 홈페이지에서 관광지 카드 하나 클릭
2. 상세페이지로 이동
3. 페이지를 아래로 스크롤하여 "위치" 섹션 확인

---

## 2. 마커 및 인포윈도우 확인

### 2.1 마커 표시 확인
- [ ] 지도 위에 빨간색 핀 모양 마커가 1개 표시되는지 확인
- [ ] 마커가 관광지의 실제 위치에 표시되는지 확인 (주소와 대략 일치하는지)

### 2.2 인포윈도우 확인
- [ ] 페이지 로드 시 마커 위에 인포윈도우가 자동으로 열려있는지 확인
- [ ] 인포윈도우에 관광지명이 표시되는지 확인
- [ ] 인포윈도우에 주소가 표시되는지 확인
- [ ] 마커를 클릭했을 때 인포윈도우가 열리는지 확인 (이미 열려있으면 닫았다가 다시 열어보기)

**확인 방법:**
1. 상세페이지의 지도 섹션 확인
2. 마커 클릭하여 인포윈도우 열기/닫기 테스트

---

## 3. 길찾기 버튼 확인

### 3.1 버튼 표시 확인
- [ ] 지도 아래에 "길찾기" 버튼이 표시되는지 확인
- [ ] 버튼에 네비게이션 아이콘(화살표)이 표시되는지 확인

### 3.2 버튼 동작 확인
- [ ] "길찾기" 버튼을 클릭했을 때 새 창(또는 새 탭)이 열리는지 확인
- [ ] 새 창에서 네이버 지도 페이지가 열리는지 확인
- [ ] 네이버 지도에서 해당 관광지 위치가 표시되는지 확인

**확인 방법:**
1. 지도 아래 "길찾기" 버튼 클릭
2. 새 창이 열리는지 확인
3. 네이버 지도 페이지가 정상적으로 로드되는지 확인

---

## 4. 좌표 정보 토글 확인

### 4.1 토글 버튼 확인
- [ ] 지도 아래에 "좌표 보기" 버튼이 표시되는지 확인
- [ ] 버튼에 눈 아이콘(Eye)이 표시되는지 확인

### 4.2 좌표 표시/숨김 확인
- [ ] "좌표 보기" 버튼을 클릭했을 때 좌표가 표시되는지 확인
- [ ] 좌표가 숫자 형태로 표시되는지 확인 (예: 37.566500, 126.978000)
- [ ] 좌표가 표시된 상태에서 버튼이 "좌표 숨기기"로 변경되는지 확인
- [ ] 버튼 아이콘도 변경되는지 확인 (눈 아이콘 → 눈에 X 아이콘)
- [ ] "좌표 숨기기" 버튼을 클릭했을 때 좌표가 사라지는지 확인
- [ ] 버튼이 다시 "좌표 보기"로 변경되는지 확인

**확인 방법:**
1. "좌표 보기" 버튼 클릭 → 좌표 표시 확인
2. "좌표 숨기기" 버튼 클릭 → 좌표 숨김 확인
3. 여러 번 토글하여 정상 작동 확인

---

## 5. 좌표가 없는 경우 확인

### 5.1 지도 섹션 숨김 확인
- [ ] 좌표 정보가 없는 관광지 상세페이지에서 지도 섹션이 표시되지 않는지 확인
- [ ] 다른 섹션(기본 정보, 이미지 갤러리, 운영 정보)은 정상적으로 표시되는지 확인

**확인 방법:**
1. 여러 관광지를 클릭하여 확인
2. 좌표가 없는 관광지가 있으면 해당 페이지에서 지도 섹션이 없는지 확인

---

## 6. 반응형 디자인 확인

### 6.1 모바일 화면 확인
- [ ] 모바일 화면(브라우저 창을 좁게 조정)에서 지도가 정상적으로 표시되는지 확인
- [ ] 지도 높이가 적절한지 확인 (너무 작거나 크지 않음)
- [ ] "길찾기" 버튼과 "좌표 보기" 버튼이 세로로 배치되는지 확인
- [ ] 버튼들이 화면 너비에 맞게 표시되는지 확인

### 6.2 데스크톱 화면 확인
- [ ] 데스크톱 화면(브라우저 창을 넓게 조정)에서 지도가 정상적으로 표시되는지 확인
- [ ] 지도 높이가 모바일보다 더 큰지 확인
- [ ] "길찾기" 버튼과 "좌표 보기" 버튼이 가로로 배치되는지 확인

**확인 방법:**
1. 브라우저 개발자 도구(F12) 열기
2. 반응형 디자인 모드 활성화 (Ctrl+Shift+M 또는 디바이스 아이콘 클릭)
3. 모바일 크기(예: iPhone 12)로 조정하여 확인
4. 데스크톱 크기로 조정하여 확인

---

## 7. 로딩 상태 확인

### 7.1 지도 로딩 확인
- [ ] 페이지를 처음 로드할 때 "지도를 불러오는 중..." 메시지가 잠깐 표시되는지 확인
- [ ] 로딩 중 스피너 아이콘이 표시되는지 확인
- [ ] 로딩이 완료되면 지도가 정상적으로 표시되는지 확인

**확인 방법:**
1. 상세페이지 새로고침 (F5)
2. 지도 섹션 영역을 주시하며 로딩 메시지 확인

---

## 8. 에러 처리 확인 (선택 사항)

### 8.1 네이버 지도 API 오류 시나리오
- [ ] 네이버 지도 API가 로드되지 않을 때 에러 메시지가 표시되는지 확인
- [ ] "새로고침" 버튼이 표시되는지 확인
- [ ] "새로고침" 버튼을 클릭했을 때 페이지가 새로고침되는지 확인

**참고:** 이 항목은 실제로 네이버 지도 API 오류가 발생해야 확인할 수 있으므로, 정상 환경에서는 확인이 어려울 수 있습니다.

---

## 9. 섹션 배치 순서 확인

### 9.1 섹션 순서 확인
- [ ] 상세페이지에서 섹션이 다음 순서로 배치되는지 확인:
  1. 기본 정보 (관광지명, 이미지, 주소 등)
  2. 이미지 갤러리 (이미지가 있는 경우)
  3. **위치 (지도 섹션)** ← 새로 추가된 섹션
  4. 운영 정보 (운영시간, 휴무일 등)

**확인 방법:**
1. 상세페이지에서 위에서 아래로 스크롤
2. 섹션 순서가 위와 일치하는지 확인

---

## 10. 여러 관광지로 테스트

### 10.1 다양한 관광지 확인
- [ ] 다른 관광지 상세페이지에서도 지도가 정상적으로 표시되는지 확인
- [ ] 각 관광지의 마커 위치가 실제 주소와 일치하는지 확인
- [ ] 모든 관광지에서 길찾기 버튼이 정상 작동하는지 확인

**확인 방법:**
1. 홈페이지에서 여러 관광지 클릭
2. 각 상세페이지에서 지도 섹션 확인
3. 길찾기 버튼 클릭하여 정상 작동 확인

---

## 빠른 확인 체크리스트 (요약)

> 시간이 부족할 때 빠르게 확인할 핵심 항목들입니다.

### 필수 확인 항목 (5분)
- [ ] **지도가 표시되는가?** - 상세페이지에서 "위치" 섹션 확인
- [ ] **마커가 보이는가?** - 지도 위에 빨간색 핀 확인
- [ ] **길찾기 버튼이 작동하는가?** - 버튼 클릭 시 네이버 지도 새 창 열림 확인
- [ ] **좌표 토글이 작동하는가?** - "좌표 보기" 클릭 시 좌표 표시/숨김 확인
- [ ] **반응형이 작동하는가?** - 브라우저 창 크기 조정하여 모바일/데스크톱 확인

### 추가 확인 항목 (10분)
- [ ] **인포윈도우가 표시되는가?** - 마커 클릭 시 관광지명과 주소 표시 확인
- [ ] **섹션 순서가 맞는가?** - 기본 정보 → 이미지 → **지도** → 운영 정보 순서 확인
- [ ] **여러 관광지에서 작동하는가?** - 다른 관광지 상세페이지에서도 정상 작동 확인

---

## 확인 완료 후 체크

모든 항목을 확인한 후:
- [ ] 정상 작동하는 항목: ✅ 표시
- [ ] 문제가 있는 항목: ❌ 표시하고 문제 내용 기록
- [ ] 확인하지 못한 항목: ⏸️ 표시하고 이유 기록

---

## 문제 발생 시 확인 사항

만약 지도가 표시되지 않거나 오류가 발생한다면:

1. **브라우저 콘솔 확인**
   - F12 키를 눌러 개발자 도구 열기
   - Console 탭에서 에러 메시지 확인
   - 에러 메시지를 스크린샷 또는 복사하여 저장

2. **환경변수 확인**
   - `.env` 파일에 `NEXT_PUBLIC_NAVER_MAP_CLIENT_ID`가 설정되어 있는지 확인
   - 서버를 재시작했는지 확인

3. **네이버 지도 API 확인**
   - 네이버 클라우드 플랫폼에서 지도 API 서비스가 활성화되어 있는지 확인
   - API 키가 유효한지 확인

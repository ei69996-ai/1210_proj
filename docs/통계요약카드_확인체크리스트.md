# 통계 요약 카드 수동 확인 체크리스트

> 개발된 통계 요약 카드 기능을 서버 실행 후 브라우저에서 직접 확인하는 체크리스트입니다.

## 📋 확인 전 준비사항

1. **서버 실행**
   ```bash
   pnpm dev
   ```
2. **브라우저에서 접속**
   - `http://localhost:3000/stats` 페이지로 이동
   - 또는 네비게이션 메뉴에서 "통계" 링크 클릭

---

## ✅ 상세 확인 항목

### 1. 페이지 접근 확인

- [ ] `/stats` 페이지에 정상적으로 접근 가능한가?
- [ ] 페이지 제목 "통계 대시보드"가 보이는가?
- [ ] 부제목 "전국 관광지 현황을 한눈에 확인하세요"가 보이는가?
- [ ] 페이지가 에러 없이 로드되는가?

### 2. 통계 요약 카드 섹션 확인

- [ ] "요약 통계" 섹션 제목이 보이는가?
- [ ] 통계 요약 카드 영역이 표시되는가?

### 3. 전체 관광지 수 카드 확인

- [ ] 첫 번째 카드에 🌐 (Globe) 아이콘이 보이는가?
- [ ] "전체 관광지" 라벨이 보이는가?
- [ ] 큰 숫자로 전체 관광지 수가 표시되는가? (예: "1,234,567개")
- [ ] 숫자가 천 단위로 구분되어 표시되는가? (예: "1,234" 형식)
- [ ] 숫자 뒤에 "개" 단위가 표시되는가?
- [ ] 카드에 테두리(border)와 배경색이 적용되어 있는가?

### 4. Top 3 지역 카드 확인

- [ ] 두 번째 카드에 🏆 (Award) 아이콘이 보이는가?
- [ ] "1위 지역" 라벨이 보이는가?
- [ ] 지역명이 표시되는가? (예: "서울", "부산", "제주")
- [ ] 해당 지역의 관광지 개수가 표시되는가? (예: "12,345개")
- [ ] 세 번째 카드에 "2위 지역"이 표시되는가?
- [ ] 네 번째 카드에 "3위 지역"이 표시되는가?
- [ ] 각 지역 카드에 순위가 올바르게 표시되는가? (1위, 2위, 3위)

### 5. Top 3 타입 카드 확인

- [ ] 다섯 번째 카드에 🏷️ (Tag) 아이콘이 보이는가?
- [ ] "1위 타입" 라벨이 보이는가?
- [ ] 타입명이 표시되는가? (예: "관광지", "음식점", "숙박")
- [ ] 해당 타입의 관광지 개수가 표시되는가? (예: "5,678개")
- [ ] 여섯 번째 카드에 "2위 타입"이 표시되는가?
- [ ] 일곱 번째 카드에 "3위 타입"이 표시되는가?
- [ ] 각 타입 카드에 순위가 올바르게 표시되는가? (1위, 2위, 3위)

### 6. 마지막 업데이트 시간 카드 확인

- [ ] 마지막 카드에 🕐 (Clock) 아이콘이 보이는가?
- [ ] "마지막 업데이트" 라벨이 보이는가?
- [ ] 상대 시간이 표시되는가? (예: "방금 전", "5분 전", "1시간 전")
- [ ] 절대 시간도 함께 표시되는가? (예: "2025-01-15 14:30")
- [ ] 시간 형식이 한국어로 표시되는가?

### 7. 카드 디자인 및 레이아웃 확인

- [ ] 모든 카드가 동일한 스타일로 표시되는가?
- [ ] 카드 간격이 적절한가?
- [ ] 카드에 테두리와 둥근 모서리가 적용되어 있는가?
- [ ] 카드 배경색이 적용되어 있는가?
- [ ] 카드에 마우스를 올렸을 때 그림자 효과가 나타나는가? (호버 효과)

### 8. 반응형 디자인 확인

#### 모바일 화면 (브라우저 개발자 도구로 확인)
- [ ] 모바일 화면에서 카드가 1열로 세로로 배치되는가?
- [ ] 카드가 화면 너비에 맞게 조정되는가?
- [ ] 텍스트가 잘리지 않고 모두 보이는가?

#### 태블릿 화면
- [ ] 태블릿 화면에서 카드가 2열로 배치되는가?
- [ ] 카드 크기가 적절한가?

#### 데스크톱 화면
- [ ] 데스크톱 화면에서 카드가 4열로 가로로 배치되는가?
- [ ] 모든 카드가 한 줄에 보이는가?
- [ ] 카드 간격이 적절한가?

### 9. 로딩 상태 확인

> ⚠️ 로딩 상태를 확인하려면 네트워크 속도를 느리게 설정하거나, 개발자 도구에서 네트워크 속도 제한을 설정해야 합니다.

- [ ] 페이지 로딩 중에 회색 박스(Skeleton UI)가 표시되는가?
- [ ] 로딩 중에도 카드 레이아웃이 유지되는가?
- [ ] 데이터가 로드되면 Skeleton UI가 사라지고 실제 데이터가 표시되는가?

### 10. 데이터 정확성 확인

- [ ] 전체 관광지 수가 0보다 큰 숫자로 표시되는가?
- [ ] Top 3 지역이 실제로 관광지 개수가 많은 순서대로 정렬되어 있는가?
- [ ] Top 3 타입이 실제로 관광지 개수가 많은 순서대로 정렬되어 있는가?
- [ ] 각 지역/타입의 관광지 개수가 합리적인 숫자인가? (너무 작거나 큰 숫자가 아닌지)

### 11. 에러 처리 확인

> ⚠️ 에러 상태를 확인하려면 API 키를 잘못 설정하거나 네트워크를 끊어야 합니다.

- [ ] API 호출 실패 시 에러 메시지가 표시되는가?
- [ ] 에러 메시지가 사용자에게 이해하기 쉬운가?
- [ ] 에러 발생 시에도 페이지가 깨지지 않고 정상적으로 표시되는가?

### 12. 접근성 확인 (선택 사항)

- [ ] 키보드로 탭 이동이 가능한가?
- [ ] 스크린 리더가 카드 내용을 읽을 수 있는가? (시각 장애인용)
- [ ] 카드에 적절한 ARIA 라벨이 있는가? (개발자 도구로 확인)

   - [ ] 전체 관광지 수가 숫자로 표시되는가?

## 💡 확인 팁

1. **개발자 도구 사용**
   - `F12` 또는 `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)로 개발자 도구 열기
   - Console 탭에서 에러 메시지 확인
   - Network 탭에서 API 호출 상태 확인

2. **반응형 확인**
   - 브라우저 개발자 도구에서 디바이스 모드 활성화 (Ctrl+Shift+M)
   - 모바일, 태블릿, 데스크톱 화면 크기로 전환하여 확인

3. **로딩 상태 확인**
   - 개발자 도구 → Network 탭 → Throttling 설정 → "Slow 3G" 선택
   - 페이지 새로고침하여 로딩 상태 확인

4. **에러 상태 확인**
   - `.env` 파일에서 `TOUR_API_KEY`를 임시로 잘못된 값으로 변경
   - 페이지 새로고침하여 에러 메시지 확인
   - 확인 후 원래 값으로 복구

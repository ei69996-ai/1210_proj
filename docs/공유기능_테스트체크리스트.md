## 2. URL 복사 기능 확인

### 2.1 기본 복사 동작
- [ ] 공유 버튼 클릭
- [ ] 토스트 알림에 "URL이 복사되었습니다." 메시지가 나타나는지 확인
- [ ] 버튼 아이콘이 Share2 → Check로 변경되는지 확인
- [ ] 버튼 텍스트가 "공유" → "복사됨"으로 변경되는지 확인
- [ ] 2초 후 버튼이 원래 상태로 돌아오는지 확인

### 2.2 복사된 URL 확인
- [ ] 클립보드에 복사된 URL 확인 (Ctrl+V 또는 Cmd+V)
- [ ] 복사된 URL이 `http://localhost:3000/places/[contentId]` 형식인지 확인
- [ ] URL이 정확하게 복사되었는지 확인

### 2.3 다양한 관광지에서 테스트
- [ ] 다른 관광지 상세페이지에서도 동일하게 동작하는지 확인
- [ ] contentId가 다른 여러 관광지에서 테스트
- [ ] 각 관광지의 URL이 올바르게 복사되는지 확인

## 4. Open Graph 메타태그 확인

### 4.1 페이지 소스 확인
- [ ] 상세페이지에서 우클릭 → "페이지 소스 보기" (또는 Ctrl+U)
- [ ] HTML 소스에서 `<meta property="og:title"` 태그가 있는지 확인
- [ ] `<meta property="og:description"` 태그가 있는지 확인
- [ ] `<meta property="og:image"` 태그가 있는지 확인
- [ ] `<meta property="og:url"` 태그가 있는지 확인
- [ ] `<meta property="og:type" content="website"` 태그가 있는지 확인

### 4.2 메타태그 내용 확인
- [ ] `og:title`에 관광지명이 올바르게 들어가 있는지 확인
- [ ] `og:description`에 관광지 설명이 100자 이내로 잘려있는지 확인
- [ ] `og:image`에 대표 이미지 URL이 들어가 있는지 확인
- [ ] `og:url`에 상세페이지 URL이 올바르게 들어가 있는지 확인

### 4.3 소셜 미디어 공유 미리보기 (선택 사항)
- [ ] [Facebook Sharing Debugger](https://developers.facebook.com/tools/debug/) 사용
  - [ ] URL 입력 후 "디버그" 클릭
  - [ ] 미리보기에서 관광지명, 설명, 이미지가 올바르게 표시되는지 확인
- [ ] [Twitter Card Validator](https://cards-dev.twitter.com/validator) 사용 (선택 사항)
  - [ ] URL 입력 후 미리보기 확인

## 테스트 시 주의사항

1. **클립보드 API**: HTTPS 환경에서만 완전히 동작합니다. `http://localhost`에서는 대체 방법이 사용됩니다.
2. **메타태그**: 소셜 미디어 플랫폼은 캐시를 사용하므로, 메타태그를 수정한 후 즉시 반영되지 않을 수 있습니다.
3. **이미지 URL**: 외부 이미지 URL이 CORS 정책에 의해 차단될 수 있습니다. 이는 정상적인 동작입니다.
